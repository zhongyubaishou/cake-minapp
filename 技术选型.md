# 《蛋糕定制》微信小程序技术选型方案

## 一、总体技术栈

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| **用户端小程序** | uni-app + Vue 3 + TypeScript | 面向微信小程序发布，保留后续扩展 H5/App 的空间；微信登录、手机号授权、定位、支付仍使用微信小程序能力 |
| **后台管理端** | Vue 3 + Element Plus | PC 端管理后台，操作效率高 |
| **后端框架** | Node.js + Koa + TypeScript | 全栈 JavaScript，开发效率高，类型安全 |
| **数据库** | MySQL 8.0 | 关系型数据为主，支持 JSON 字段与地理距离计算 |
| **ORM** | Prisma | 类型安全的数据库操作，自动生成类型，Migration 管理 |
| **文件存储** | 本地存储 + 静态资源服务 | V1 阶段使用本地文件系统，后续可扩展腾讯云 COS |
| **API 风格** | RESTful | 统一 `{ code, message, data }` 响应格式 |
| **状态管理（用户端）** | Pinia | Vue 3 官方推荐，模块化状态管理 |
| **CSS 预处理器（用户端）** | SCSS | 变量、嵌套、mixin 支持 |
| **CSS 方案（后台端）** | CSS 变量 + Element Plus 主题覆盖 | 前缀 `--cake-*`，统一视觉风格 |
| **构建工具（后台端）** | Vite | 快速开发构建，HMR |

## 二、各端详细技术说明

### 2.1 用户端小程序（userend/）

| 技术项 | 选型 |
|--------|------|
| 框架 | uni-app 3.x |
| UI 框架 | Vue 3 Composition API |
| 语言 | JavaScript（保留升级 TypeScript 空间） |
| CSS 方案 | SCSS |
| 请求库 | uni.request 封装（`utils/request.js`） |
| 状态管理 | Pinia（`stores/user.js`） |
| Mock 方案 | 本地 JSON mock（`constants/app.js` 开关控制） |
| 运行环境 | 微信小程序（通过 uni-app 编译） |

### 2.2 后台管理端（admin/）

| 技术项 | 选型 |
|--------|------|
| 框架 | Vue 3 + Vite |
| 语言 | TypeScript |
| UI 库 | Element Plus |
| 路由 | Vue Router 4 |
| 请求库 | Axios（`api/http.ts` 封装） |
| 权限控制 | 路由守卫 + localStorage Token |

### 2.3 后端（backend/）

| 技术项 | 选型 |
|--------|------|
| 运行时 | Node.js 18+ |
| 框架 | Koa 2 |
| 语言 | TypeScript |
| ORM | Prisma |
| 鉴权 | JWT（`jsonwebtoken`） |
| 参数校验 | 手动校验 + 类型定义 |
| 文件上传 | `@koa/multer` |
| 日志 | 自定义日志中间件 |
| 跨域 | `@koa/cors` |
| 微信 SDK | 微信服务端 API 直接调用 |

## 三、不引入的技术

以下技术在 V1 阶段明确不引入：

1. Redis（单机内存即可满足 V1 需求）
2. 消息队列（RabbitMQ / Kafka）
3. Docker / K8s（直接部署）
4. Nginx（V1 可 Koa 直接对外或简单 Nginx 反代）
5. 微服务架构
6. GraphQL
7. WebSocket（后台声音提醒用轮询即可）
8. 腾讯云 COS SDK（V1 本地文件存储，后续扩展）

## 四、V1 关键技术决策

| 决策项 | 决策 | 原因 |
|--------|------|------|
| 不接入微信支付 | 线下收款 | 商家暂未注册个体工商户、无微信支付商户号 |
| 退款功能代码保留但不启用 | 线下协商 | V1 用户端不展示退款入口 |
| 单店单老板 | 无需 RBAC | 管理员仅老板一人，硬编码 admin 角色 |
| 配送距离用经纬度计算 | 使用球面距离公式 | 门店固定经纬度，无需接入地图 API |
| 不使用 Nginx 反向代理 | Vite proxy + Koa 直出 | 减少运维复杂度 |
| Prisma 管理数据库 | 不用手动 SQL | 类型安全、可迁移 |

## 五、目录结构约定

```
project/
├── userend/          # uni-app 用户端小程序
│   ├── pages/        # 页面
│   ├── components/   # 组件
│   ├── stores/       # Pinia 状态
│   ├── utils/        # 工具函数、请求封装、adapter
│   ├── constants/    # 常量定义
│   └── styles/       # 全局样式
├── admin/            # Vue 3 后台管理端
│   ├── src/
│   │   ├── views/    # 页面
│   │   ├── components/ # 组件
│   │   ├── api/      # 请求封装
│   │   ├── router/   # 路由
│   │   ├── utils/    # 工具函数
│   │   └── styles/   # 全局样式
│   └── vite.config.ts
└── backend/          # Koa 后端
    ├── src/
    │   ├── routes/       # 路由层
    │   ├── controllers/  # 控制器层
    │   ├── services/     # 服务层（业务逻辑）
    │   ├── middleware/    # 中间件
    │   ├── utils/        # 工具函数
    │   └── config/       # 配置
    └── prisma/
        ├── schema.prisma # 数据模型
        └── migrations/   # 迁移文件
```

## 六、设计风格

用户端采用 **Mondrian（几何色块）+ Memphis（散落圆点）+ Swiss（等比间距）** 融合风格：

| 色彩 | 色值 | 用途 |
|------|------|------|
| 陶土橙 | `#d95c33` | 主色、按钮、强调 |
| 点缀蓝 | `#2d5fbf` | 辅助强调 |
| 点缀黄 | `#e8b830` | 辅助强调 |
| 暖白背景 | `#f6f1ed` | 页面背景 |
