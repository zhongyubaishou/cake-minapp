# 蛋糕定制微信小程序 — 数据库表设计

## 一、概述

共设计 **15 张核心表**，覆盖用户、管理员、门店、商品、订单、支付、售后等全部业务模块。

### E-R 关系概览

```
admin_user（后台管理员，单店老板账号）
user ──< orders ──> order_customization (1:1)
user ──< user_favorite >── cake_case
orders ──< payment_records
orders ──< refund_records
orders ──< order_operation_logs
orders ──< after_sale
orders ──< delivery
store <──> (全局单条配置)
category ──< products
```

## 二、表结构设计

### 2.1 admin_user（后台管理员表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 管理员 ID |
| username | VARCHAR(50) UNIQUE NOT NULL | 登录账号 |
| password_hash | VARCHAR(255) NOT NULL | bcrypt 加密密码 |
| nickname | VARCHAR(50) | 显示昵称 |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | 更新时间 |

> V1 阶段仅一个管理员账号（老板），预置 seed 数据。

### 2.2 user（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 用户 ID |
| openid | VARCHAR(100) UNIQUE NOT NULL | 微信 OpenID |
| unionid | VARCHAR(100) | 微信 UnionID |
| nickname | VARCHAR(100) | 微信昵称 |
| avatar_url | VARCHAR(500) | 微信头像 URL |
| phone | VARCHAR(20) | 手机号（V1 非必填） |
| default_address | JSON | 默认配送地址（经纬度+详细地址） |
| created_at | DATETIME DEFAULT NOW() | 注册时间 |
| updated_at | DATETIME | 更新时间 |

### 2.3 store（门店配置表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 配置 ID（单条记录） |
| name | VARCHAR(100) NOT NULL | 门店名称 |
| address | VARCHAR(300) NOT NULL | 门店地址 |
| latitude | DECIMAL(10, 7) | 门店纬度 |
| longitude | DECIMAL(10, 7) | 门店经度 |
| business_start | TIME DEFAULT '06:00' | 营业开始时间 |
| business_end | TIME DEFAULT '18:00' | 营业结束时间 |
| delivery_range_km | DECIMAL(5, 2) DEFAULT 5.00 | 配送范围（km） |
| delivery_fee_per_km | DECIMAL(5, 2) DEFAULT 2.00 | 每公里配送费（元） |
| min_advance_hours | INT DEFAULT 6 | 最早提前预约小时数 |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 2.4 category（商品分类表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 分类 ID |
| name | VARCHAR(50) NOT NULL | 分类名称 |
| sort_order | INT DEFAULT 0 | 排序 |
| is_enabled | TINYINT(1) DEFAULT 1 | 是否启用 |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | 更新时间 |

默认分类：生日蛋糕、儿童蛋糕、定制蛋糕。

### 2.5 product（商品表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 商品 ID |
| category_id | INT NOT NULL | 所属分类 |
| name | VARCHAR(100) NOT NULL | 商品名称 |
| image_url | VARCHAR(500) | 主图 URL |
| base_price | DECIMAL(10, 2) NOT NULL | 基础价格（2 磅） |
| description | TEXT | 商品描述 |
| flavors | JSON | 可选口味列表 |
| theme_styles | JSON | 可选主题风格列表 |
| is_on_sale | TINYINT(1) DEFAULT 1 | 是否上架 |
| sort_order | INT DEFAULT 0 | 排序 |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 2.6 cake_case（蛋糕案例表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 案例 ID |
| title | VARCHAR(100) NOT NULL | 案例标题 |
| image_url | VARCHAR(500) NOT NULL | 案例图片 |
| theme | VARCHAR(50) | 主题风格 |
| description | TEXT | 案例描述 |
| is_visible | TINYINT(1) DEFAULT 1 | 是否展示 |
| sort_order | INT DEFAULT 0 | 排序 |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 2.7 user_favorite（用户收藏表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 收藏 ID |
| user_id | INT NOT NULL | 用户 ID |
| case_id | INT NOT NULL | 案例 ID |
| created_at | DATETIME DEFAULT NOW() | 收藏时间 |

UNIQUE KEY: `(user_id, case_id)`

### 2.8 orders（订单表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 订单 ID |
| order_no | VARCHAR(32) UNIQUE NOT NULL | 订单编号 |
| user_id | INT NOT NULL | 用户 ID |
| product_id | INT | 关联商品 ID |
| category_id | INT | 分类 ID |
| pickup_type | ENUM('PICKUP', 'DELIVERY') NOT NULL | 取货方式 |
| status | VARCHAR(30) NOT NULL | 订单状态（见状态机设计） |
| pay_status | VARCHAR(20) DEFAULT 'UNPAID' | 支付状态 |
| payment_method | VARCHAR(20) | 支付方式（OFFLINE） |
| pickup_time | DATETIME NOT NULL | 预约取货/送达时间 |
| product_amount | DECIMAL(10, 2) | 商品金额（老板确认） |
| delivery_fee | DECIMAL(10, 2) | 配送费 |
| total_amount | DECIMAL(10, 2) | 订单总金额 |
| receiver_name | VARCHAR(50) | 收货人姓名 |
| receiver_phone | VARCHAR(20) | 收货人手机号 |
| delivery_address | VARCHAR(500) | 配送详细地址 |
| delivery_lat | DECIMAL(10, 7) | 配送地址纬度 |
| delivery_lng | DECIMAL(10, 7) | 配送地址经度 |
| delivery_distance | DECIMAL(5, 2) | 配送距离（km） |
| is_overtime | TINYINT(1) DEFAULT 0 | 是否非营业时间预约 |
| admin_remark | TEXT | 老板备注 |
| user_remark | TEXT | 用户备注 |
| cancel_reason | VARCHAR(300) | 取消原因 |
| accepted_at | DATETIME | 接单时间 |
| paid_at | DATETIME | 支付时间 |
| completed_at | DATETIME | 完成时间 |
| canceled_at | DATETIME | 取消时间 |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 2.9 order_customization（订单定制信息表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 定制 ID |
| order_id | INT UNIQUE NOT NULL | 关联订单（1:1） |
| product_weight | DECIMAL(5, 2) DEFAULT 2.00 | 购买磅数 |
| actual_weight | DECIMAL(5, 2) | 实际制作磅数（×1.5） |
| flavor | VARCHAR(50) DEFAULT '原味' | 口味 |
| theme_style | VARCHAR(50) DEFAULT '成人' | 主题风格 |
| blessing_text | VARCHAR(200) | 祝福文字 |
| plate_count | INT DEFAULT 0 | 碟子数量 |
| candle_count | INT DEFAULT 0 | 蜡烛数量 |
| reference_image_url | VARCHAR(500) | 参考图片 URL |
| case_id | INT | 关联案例 ID（按案例定制） |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |

### 2.10 payment_record（支付记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 支付记录 ID |
| order_id | INT NOT NULL | 关联订单 |

| amount | DECIMAL(10, 2) | 支付金额 |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |

### 2.11 refund_record（退款记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PRIMARY KEY AUTO_INCREMENT | 退款记录 ID |
| order_id | INT NOT NULL | 关联订单 |
| refund_no | VARCHAR(32) UNIQUE | 退款编号 |
| amount | DECIMAL(10, 2) | 退款金额 |
| reason | TEXT | 退款原因 |
| status | VARCHAR(20) | 退款状态 |
| created_at | DATETIME DEFAULT NOW() | 创建时间 |
| updated_at | DATETIME | 更新时间 |

> V1 暂不启用线上退款，表结构保留。

## 三、索引建议

| 表 | 索引 | 类型 |

| user | openid | UNIQUE |
| orders | user_id, status, order_no | INDEX |
| orders | pickup_time | INDEX |
| orders | created_at | INDEX |
| user_favorite | (user_id, case_id) | UNIQUE |
| cake_case | is_visible, sort_order | INDEX |
| product | category_id, is_on_sale | INDEX |
| order_operation_log | order_id, created_at | INDEX |
| after_sale | order_id, user_id | INDEX |

## 四、Prisma Schema 约定

- 表名：snake_case（`cake_case`、`after_sale`）
- 字段名：snake_case
- 关联：使用 Prisma 关系定义
- 时间字段：`created_at`、`updated_at` 统一命名
- 枚举：Prisma Enum 定义状态字段
- JSON 字段：用于口味列表、主题风格列表等灵活配置
