# 🎂 蛋糕定制微信小程序

面向小型蛋糕门店的微信预约小程序。用户在线浏览商品和案例，提交蛋糕定制需求；商家通过后台管理端确认报价、管理订单状态流转、处理配送/自提及售后反馈。

---

## 🧩 项目架构

```
project/
├── userend/          # 微信小程序用户端 (uni-app + Vue 3)
├── admin/            # PC 后台管理端 (Vue 3 + Element Plus + TypeScript)
├── backend/          # 后端 API 服务 (Node.js + Koa + TypeScript + Prisma)
└── server/           # 数据库 Schema (Prisma + MySQL 8.0)
```

---

## 🛠 技术栈

| 层级 | 技术 |
|------|------|
| 用户端 | uni-app 3.x + Vue 3 + SCSS + Pinia |

| 测试 | Vitest + Supertest |
| 构建 / 运行 | Vite（后台）/ tsx watch（后端）/ HBuilderX CLI（用户端） |

## ✨ 功能概览

### 用户端（微信小程序）

- 微信登录 & 手机号授权
- 首页门户（门店信息、分类入口、案例入口）
- 商品浏览（按分类筛选、商品详情）
- 蛋糕定制下单（磅数、口味、主题、祝福文字、碟子蜡烛、参考图上传）
- 取货方式（到店自提 / 商家配送）
- 配送距离自动计算（微信内置定位）
- 蛋糕案例图库（浏览、收藏、按案例定制）
- 我的订单（列表、状态追踪）
- 售后反馈（完成后 3 天内可提交）

### 后台管理端

- 管理员登录
- 工作台（今日订单、销售额概览）
- 订单管理（查看定制需求、确认报价/接单、状态流转）
- 商品管理（新增/编辑/上下架/排序）
- 分类管理（增删改查/启用停用）
- 蛋糕案例管理（图片库维护）
- 配送管理（待配送列表、开始配送、确认送达）
- 售后处理（查看反馈、标记已处理）
- 数据报表
- 门店设置
- 新订单声音提醒

### 后端 API

- 15 张数据库表完整覆盖业务模块
- 订单状态机严格校验
- 经纬度球面距离计算
- 文件上传
- 完整测试覆盖

## 🚀 快速启动

```bash
# 1. 启动后端 API 服务
cd backend
npm install
npm run db:migrate   # 初始化数据库（需先配置 .env 中的 DATABASE_URL）
npm run dev          # 监听 http://localhost:3000

# 2. 启动后台管理端
cd admin
npm install
npm run dev          # 监听 http://localhost:5173

# 3. 启动用户端（需安装 HBuilderX CLI）
cd userend
npm run dev:mp-weixin
```

## 🔄 订单状态流转

```
用户提交订单
  └── 待接单
        └── 商家确认报价/接单
              └── 已支付/待制作
                    └── 商家点击开始制作
                          └── 制作中
                                ├── 自提：待自提 → 已完成
                                └── 配送：待配送 → 配送中 → 已完成
```

## 📦 V1 版本说明

当前为首版 MVP，已完成 5 周开发周期的核心功能。

### 关键设计决策

| 决策 | 说明 |
|------|------|
| 不接入微信支付 | 商家暂未注册个体工商户，支付走线下确认模式 |
| 单店单老板 | 仅一个管理员账号，无需 RBAC 权限系统 |
| 线下收款 | 商家确认报价后订单直接进入"已支付/待制作"状态 |
| 不启用线上退款 | 退款由商家与用户线下协商处理 |
| 不接入第三方配送 | 商家自配送 |
| 不引入 Redis / Docker | V1 保持架构简洁 |

### 配送规则

- 配送范围：门店周边 5km 内
- 配送费：`ROUND(距离) × 2` 元/公里
- 预约规则：至少提前 6 小时

### 不开发范围（V1）

会员、优惠券、储值卡、积分、拼团、秒杀、满减、多门店、多员工权限、第三方跑腿、发票、打印小票、自动售后退款
